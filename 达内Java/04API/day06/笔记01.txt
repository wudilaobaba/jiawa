# XML
## 什么是XML
可扩展的标记语言
XML的用途：
  1.统一的数据文件规范，统一
  2.统一的解析API，易于使用
  3.使用非常广泛

eclipse中new-->other--XML--XML File--Next--重命名--finish

### XML文件是一种文本文件。
  -一般情况下都是UTF-8编码，支持国际化。

### 处理指令
是一个可选指令，用于指示XML解析器(XML API)如何处理XML文件。一般情况下都会写在XML文件的第一行
<?xml version="1.0" encoding="UTF-8"?>
- 处理指令的起始符号和结束符号<?xml ?>
- version="1.0" 用于指定XML版本，至今一直是1.0
- encoding="UTF-8" 是指XML中文字的编码，建议使用UTF-8

### 标记 tag
标记：由`<`开始，由`>`结束，中间是标记名
标记名可以任意扩展，建议使用英文
例如：
  <books>
标记分为：
- 开始标记
    <books>
- 结束标记
    </books>
- 标记名区别大小写
- XML规定：标记必须成对使用
- XML规定：XML只能有一个唯一的根标记
- 标签的嵌套可以任意扩展

### 属性 attribute
在开始标记上可以定义属性
  属性名 = "属性值"
- 属性名可以任意扩展，但不能重复
  - 一般情况下都是用英文属性名
- 属性值可以任意扩展。必须使用引号
- 在开始标记中可以定义多个属性，属性没有顺序

### 内容 content
开始标记和结束标记之间的称为内容
1.文本内容
2.子标记
3.子标记和文本混合体

### 元素 element
元素:开始标记+内容+结束标记 的整体
标记的名就是元素的名
- 可以有空元素：没有内容的元素，可以省略为"自结束标记": <name/>

### 注释
  <!-- 作者 -->

### XML语法是如何"可以扩展"
1.标记名(元素名)可以任意扩展
2.标记的嵌套关系可以任意扩展，元素的嵌套关系可以任意扩展
3.元素的属性可以任意扩展，没有限制
